Interface:

	master.pd

Receiver: master
Possible Messages:
	
	new							Spawn a new note
	reset 						Delete all notes

ALL RANGES ARE FROM 0-127

I recommend you use the following effects for Play Mode sliders (in this order!):
Note that all have "auto" methods except phase, which just takes a float (0-127).

	phase
	chorus
	ringmod
	delay
	tremolo
	filter

Receiver: effects
Possible Messages:

	// all auto methods will automatically turn effects on & off (0 == off)

	chorus auto <float>
	chorus on <float>			0 or 1
	chorus rate <float>								
	chorus depth <float>												
	chorus mix <float>				 									

	tremolo auto <float>
	tremolo on <float> 			0 or 1
	tremolo rate <float> 							
	tremolo depth <float> 				
	tremolo shape <float> 		0 - sine, 
								1 - sawtooth, 
								2 - reverse sawtooth, 
								3 - square

	delay auto <float>
	delay on <float> 			0 or 1
	delay feedback <float>		 							
	delay time <float> 			 								
	delay amount <float> 						

	ringmod auto <float>
	ringmod on <float> 			0 or 1
	ringmod rate <float> 							
	ringmod depth <float> 											

	filter auto <float>
	filter on <float> 			0 or 1
	filter frequency <float> 	 		
	filter resonance <float> 				
	filter gain <float> 		Any number								
	filter type <symbol>		"lowpass", 		// No effect from Gain
								"bandpass", 	// No effect from Gain
								"highpass",		// No effect from Gain
								"notch", 		// No effect from Gain
								"peak", 		// Gain effects this
								"highshelf", 	// Gain effects this
								"lowshelf",		// Gain effects this

	pan <float> 				0 = left, 64 = center, 127 = right 
	volume <float> 				


Receiver: note#  (e.g. "note1", "note2", etc) 
Possible messages:  

	<list> 						(key, velocity) pair as a list, both from 0 to 127
	pitchbend <float>
	pitchbend_range <float>
	adsr <list>					"attack, decay, sustain, release",  
	waveform <symbol>			name of waveform. see bottom for names
	flush <bang>				sends note-off to all notes
	print <float>				0 or 1 - Prints adsr note-on & offs in Pd - debugging
	phase <float>				Phase offset
	mode <float>				0 - Normal (phase does nothing)
								1 - Subtraction (subtracting wave with phase offset)
								2 - Multiplication
	filter frequency <float>	set lowpass filter frequency
	volume <float> 				set volume
